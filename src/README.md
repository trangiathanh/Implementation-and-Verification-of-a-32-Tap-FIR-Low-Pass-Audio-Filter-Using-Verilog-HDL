\# DESIGN AND SIMULATION OF 32-TAP FIR LOW-PASS FILTER

\*(Note: This project strictly focuses on system simulation using MATLAB/Simulink and ModelSim, and has not yet been implemented on an FPGA hardware board.)\*

\*\*I/O Signal List in Simulation Environment\*\*

\#\# Table of Contents  
\- \[fir32\_lowpass\](\#fir32\_lowpass)  
\- \[fir\_tb\](\#fir\_tb)

\---

\#\# fir32\_lowpass

File: \`fir32\_lowpass.v\`

\*(The core module containing the Direct Form structure of the 32-tap FIR filter. It receives simulation signals from the testbench, performs Multiply-Accumulate (MAC) operations, and outputs the result.)\*

| Signal | Direction | Type/Width | Description |  
|---|---|---|---|  
| clk | input | logic | System clock signal used for simulation. |  
| rst\_n | input | logic | Active-low reset signal. Initially set to 0 to reset the system, then driven to 1\. |  
| x\_in | input | logic | Input signal containing high-frequency noise (simulating audio data). |  
| y\_out | output | logic | Output signal after passing through the FIR filter, with high-frequency noise significantly attenuated. |

\---

\#\# fir\_tb

File: \`fir\_tb.v\`

\*(The Testbench environment on ModelSim used to stimulate the \`fir32\_lowpass\` module. This block has no external I/O ports but uses internal signals to supply simulated audio data and observe waveforms.)\*

| Signal | Direction | Type/Width | Description |  
|---|---|---|---|  
| clk | internal | logic | Clock signal generated by the testbench to drive the filter. |  
| rst\_n | internal | logic | Signal controlling the simulation reset process. |  
| x\_in | internal | logic | Noisy input signal fed into the filter by the testbench. In ModelSim, this waveform exhibits rapid, continuous fluctuations. |  
| x\_clean | internal | logic | The ideal original signal (noise-free) used as a reference benchmark. |  
| y\_out | internal | logic | The received signal from the FIR filter for waveform plotting. The graph shows a smoother wave that closely tracks \`x\_clean\`. |  
| n | internal | integer | Counter variable used to iterate through the simulation loops. |
